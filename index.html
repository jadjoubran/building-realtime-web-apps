Authenticating Users


  • Open the Pusher Documentation

    Open pusher.com/docs/authenticating_users

  • Client side Authentication endpoint

    After reading the docs, go back to your editor and open the js/app.js and fix the existing code to use the authentication under auth.php.
    Be careful with paths!

  • Handle authentication

    Open auth.php and check the source code.
    For the sake of this tutorial, we are assuming that the user is logged in and each attendee will return his own information as if he was logged in.

  • Testing Client Side Integration

    Open Chrome Developer Tools's Console then right click and enable Log XMLHTTPRequests. Then Refresh the page and make sure that the authentication process succeeded.
